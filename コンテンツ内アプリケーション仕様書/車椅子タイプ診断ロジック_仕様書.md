# 車椅子タイプ診断ロジック 仕様書（自走式／介助式）

## 1. 目的・背景

### 1.1 アプリを作る背景（ビジネス上の課題）

本アプリは、あくまで自走式・介助式のどちらにも対応する汎用の診断ツールであり、自走式を検討している人だけを対象にしたものではない。ただし、設計時に以下の傾向を踏まえておく必要がある。

- 介福本舗のECでは、介助式より自走式を購入する人が圧倒的に多い。実店舗での比率とは傾向が逆になっている。
- ECと実店舗の客層の違いを考慮しても、明らかに自走式が適していない人が自走式を購入してしまっている可能性が高い。
- 根拠：商品口コミで「思ったより大きかった」「重たかった」といった、自走式のサイズ・重さに関する不満の声が見られる。
- 福祉用具専門相談員の実感：対面でヒアリングを重ねていくと、「結局は介助式でよい」という結論に落ち着くケースが多い。
- 以上の傾向から、本アプリでは「自分で操作したい」という回答を最終結論として即採用せず、実店舗のヒアリングに近い形でリアルな使用シーンを確認するステップ（Q1-b）を、自走式方向に進んだ人に対して設ける。これは設計上の重み付け（ミスマッチが起きやすい方向により丁寧な確認を入れる）であり、アプリ全体の対象を自走式検討者に限定するものではない。介助式方向に進んだ人にも、同じく移乗の負担を確認したうえで型（標準型／多機能型）を案内する。

### 1.2 アプリの内容

「[車椅子_自走式介助式選定ガイド.md](../車椅子_自走式介助式選定ガイド.md)」内の「4. あなたはどっち？ YES/NOで分かる車椅子診断チェックリスト」は、チェックボックス形式で分かりにくいため、Web上で回答していくだけで自走式／介助式の目安が分かる簡易診断アプリに作り替える。本ドキュメントは、そのアプリの判定ロジックの詳細仕様。

判定ロジックは以下の資料をもとに設計・検証している。

- `車椅子_自走式介助式選定ガイド.md`：3大チェックポイント（①本人の身体状況／②使用環境／③介助者の状況）とYES/NOチェックリスト
- `20260715_介福本舗_コンテンツページヒアリング.txt`：福祉用具専門相談員への1回目の実務ヒアリング（現場で介助式を勧めるケースが多い理由、自走式が向くケースの実例など）
- 福祉用具専門相談員による2回目のフィードバック（2026-07-17）：初版フロー（5問構成）に対し「現場で介助式が選ばれる理由の説明に寄りすぎて、かえって分かりにくい」との指摘。判定の本質は「誰が操作するか」の一点であり、そこに移乗動作の負担で標準型／多機能型を分けるだけで十分、という趣旨の指摘を受け、ロジックを全面的に作り直した。
- keitoによるアプリ意図の共有（2026-07-17）：上記1.1のEC購入傾向・口コミ・専門相談員の実感を踏まえ、本アプリの目的を「中立な診断」から「自走式検討者へのリアリティチェック」に再定義した（本仕様書はその改訂版）。

商品カタログは楽天市場店（`https://www.rakuten.co.jp/kaifukuhonpo/`）を実際に確認した内容に基づく（2026-07-15時点）。介福本舗の取扱は手動の自走式・介助式のみで、電動車椅子の取扱はない。

### 1.3 本アプリの想定対象層（2026-07-22追記）

本アプリ、および介福本舗が取り扱う手動車椅子「よかセレクト」「よかべスター」が、そもそもどのような層を対象にした商品なのかを、社内FB（2026-07-22）を踏まえて整理する。介福本舗はフルリクライニング車椅子・電動車椅子を取り扱っておらず、これらが適する層には自社品を案内できないため、対象層の明確化は診断ロジックそのものより前段の重要な前提になる。

- **主な想定対象**：介護度で言うと要介護1〜3程度が目安（あくまで目安。下記の留意点を参照）。
- **実際のEC購入者の傾向**：要支援1〜2・非該当程度のより軽度な層も一定数利用している。普段の生活動作には大きな支障がないものの、長距離移動時の体力的な負担を軽減する目的で車椅子を使うケースがあるため。この層は本アプリの対象として問題なく、Q1-b（使用シーン確認）など既存の設計で十分対応できる。
- **自社品への誘導を避けたほうがよい層（イメージ）**：寝たきり、座位が保持できない、足の拘縮・変形があるなど、通常の座面に座って自走・介助する前提の手動車椅子（よかセレクト／よかべスター）自体が身体的に適合しない可能性が高い層。介護度で言うと要介護4〜5程度が目安。このような層には、介福本舗が取り扱っていないフルリクライニング車椅子や電動車椅子などの専門的な検討が必要になる場合があるため、案内せずに福祉用具専門相談員への相談を促すのが適切と考えられる。
- **留意点（社内FBより）**：上記はあくまで介護度に基づく目安であり、「要介護2でも寝たきりの方、要介護4でも多少歩ける方がいるため、介護度だけで一概に区切るのは危険」との指摘を受けている。そのため、将来的に診断フロー側で対象外を判定する場合も、介護度のラベルそのものではなく、具体的な身体状態（寝たきりかどうか／座位を保持できるかどうか／足の拘縮・変形の有無など）を基準にすることが望ましい。

※本節を踏まえた対応方針（2026-07-24確定）：対象外層への対応は、Q1〜Q2の判定ロジックに新しい質問・分岐として組み込むことはしない。代わりに、アプリ冒頭のリード文（4章）に「該当する場合は本診断を進めずに専門相談員へ相談してほしい」という注記を表示するのみとし、該当するかどうかの判定はユーザー自身の判断に委ねる。

## 2. 判定の設計思想（2026-07-17改訂）

初版（Q1〜Q5の5問構成）は、専門相談員への1回目ヒアリングで得た「介助式が選ばれる理由」を丁寧に反映しようとするあまり、質問数が多く分岐も複雑になっていた。2回目のフィードバックで「操作者が誰かで9割決まる、それ以外は個別事情なので事例で示せばよい」という趣旨の指摘を受け、以下の設計思想に切り替えた。さらにkeitoからのアプリ意図の共有（1.1）を受け、**設計思想の最上位に「自走式検討者へのリアリティチェック」という位置づけを明記**した。

0. **自走式・介助式どちらにも対応する汎用ツールとしたうえで、自走式方向には確認を1段厚くする**：本アプリは自走式検討者専用のツールではなく、Q1で操作者を確認したうえで自走式・介助式どちらの人にも案内する診断ツール。ただし、ECでは自走式の購入比率が実店舗より明らかに高く、「思ったより大きい・重い」という口コミが示すとおり、実際には合わない人が一定数自走式を購入してしまっているという実情があるため、Q1で「乗る人自身が操作する」を選んだ人に対しては、Q1-bで実店舗のヒアリングに近い形で現実の使用シーンを具体的に確認し、必要に応じて介助式への見直しを提案するプロセスを設ける。「常に介助者が操作する」を選んだ人はこの追加確認の対象ではないが、これはEC上でのミスマッチ傾向が観測されていないためであり、介助式の案内自体を簡略化・軽視するという意味ではない（Q2で移乗の負担を確認し、標準型／多機能型を丁寧に案内する点は自走式と同等）。
1. **核となる分岐は1つだけ**：「乗る人自身が操作するか、常に介助者が操作するか」。これで自走式／介助式の大枠が決まる。ただし0.の通り、この自己申告（特に「自分で操作したい」という回答）はそのまま鵜呑みにせず、後続のQ1-bで裏取りする。
2. **型（標準型／多機能型）は移乗動作の負担で決める**：自走式・介助式のどちらでも、「車椅子⇄ベッド・トイレなどへの乗り移りが自分で立ち上がってできるか」で標準型（よかセレクト）と多機能型（よかべスター）に分岐する。この軸は自走式・介助式で共通。
3. **「自走式を選んだが実際には介助式で十分」なケースを拾う**：自分でこぐ体力はあっても、実際の使用シーンが「通院時の付き添いだけ」など限定的で、普段はある程度歩けるという人の場合、本来は軽量な介助式の方が介助者の負担が少なく使いやすいことがある。ここを見落とすと「自走式を選んだのに実際はほとんど自分でこがず、介助者が押している」というミスマッチが起き、口コミの「思ったより大きい・重い」につながる。そのため、自走式を選んだ人だけに使用シーンを確認する補助質問（Q1-b）を挟み、単なる補足ではなくアプリの核心的な価値として扱う。
4. **自由記述・AIによる解釈は使わない**：本アプリは静的なWebページ（バックエンド・AI連携なし）で運用するため、自由記述欄を設けても内容を解釈して分岐に反映することができない。使用シーンの確認は、あらかじめ想定した選択肢（選択式）でカバーする。カバーしきれない個別事情は、既存の使用シーン別の事例紹介で補う。
5. **積み下ろし負担・トランク内寸などの個別事情は質問では聞かず、事例と補足注記で扱う**：初版のQ4・Q5（介助者の積み下ろし負担／車のトランク内寸）は、多くのケースで移乗負担の質問だけでは決まらない個別事情であり、専門相談員のフィードバックでも「その他は用途別の事例で示すとよい」とされた。そのため独立した分岐にはせず、結果画面の注記・事例として案内する。

## 3. 全体フロー（テキストツリー）

```
Q1 車椅子は誰が操作しますか？
├─ 常に介助者が操作する
│  └─ Q2 移乗（車椅子⇄ベッド・トイレなど）は自分で立ち上がってできますか？
│     ├─ できる → 【介助式・標準型】(よかセレクト 介助式)
│     └─ 立ち上がるのが難しい・大変 → 【介助式・多機能型】(よかべスター 介助式)
└─ 乗る人自身が操作する
   └─ Q1-b 車椅子を使う主な場面は？
      ├─ I. 屋内外問わず、日常的にいろいろな場面で自分の力で動きたい（買い物・散歩・庭先の移動など）
      │  └─ Q2へ（自走式のまま）
      ├─ II. 外出はほとんど付き添ってもらうが、自宅の中だけは家族を呼ばずに自分で移動したい
      │  └─ Q1-b-II 自宅の中は、車椅子を使わずに（杖や伝い歩き、手すり、スロープなどで）移動できますか？
      │     ├─ できる（車椅子がなくても室内は移動できる） → Q2へ（介助式として確定。「常に介助者が操作する」と同じ扱い）
      │     └─ できない（車椅子がないと室内移動が難しい） → Q2へ（自走式のまま。屋内での取り回しに関する注記を表示）
      ├─ III. 通院やデイサービスの送迎など、決まった外出の時だけ使う予定／普段は杖や伝い歩きなどである程度自分の足で歩ける
      │  └─ Q2へ（介助式として確定。「常に介助者が操作する」と同じ扱い）
      ├─ IV. 家族との買い物・外食・旅行などたまの外出で使う予定／普段の歩行には不安がある
      │  └─ Q2へ（介助式として確定。「常に介助者が操作する」と同じ扱い）
      └─ V. まだ使う場面がはっきり決まっていない・その他
         └─ Q2へ（自走式のまま。中立に進める）
   └─ Q2 移乗（車椅子⇄ベッド・トイレなど）は自分で立ち上がってできますか？
      ├─ できる → 【自走式・標準型】(よかセレクト 自走式)
      └─ 立ち上がるのが難しい・大変 → 【自走式・多機能型】(よかべスター 自走式)
```

Q1-bは「乗る人自身が操作する」を選んだ場合のみ表示する補助質問であり、新しい独立ゲートではなく、あくまでQ1の答えを使用シーンに照らして確認し直すためのステップという位置づけ。Q2（移乗）はQ1の結果（自走式グループ／介助式グループ）にかかわらず共通で聞く。

II（できる）・III・IVは、選択式の回答結果に基づいてアプリ側が介助式として結論を出す（ユーザーに「それでも自走式にしますか？」という再選択はさせない）。この設計判断の理由は4章冒頭を参照。

なお、介助式という結果が出た人には、上記の必須フローとは別に、結果画面で任意の「さらに自分に合う車椅子を探す」チェックリスト（4章参照）を表示する。これはQ1・Q2の分岐には含めず、あくまで結果が出たあとに希望者だけが使う補助機能。

## 4. 各質問の詳細

### アプリ冒頭のリード文（Q1の前に表示、全員共通）

「いくつかの質問に答えるだけで、自走式・介助式どちらが合うかの目安がわかります。」（自走式・介助式いずれかに偏らない、中立な案内文とする）

「寝たきりの方、座位を保つことが難しい方、足に拘縮や変形がある方など、座って使うことを前提とした一般的な車椅子が身体的に適さない可能性がある場合、本診断ではうまくご案内できないことがあります。その場合は本診断を進めずに、福祉用具専門相談員へ直接ご相談ください。」（1.3節の想定対象層を踏まえた注記。新しい質問・分岐は作らず、Q1の前に表示する文言のみで対応する。該当するかどうかの判定はユーザー自身の判断に委ね、アプリ側で選択式にはしない）

「この診断はあくまで参考です。ご心配な点がある場合は、購入前に福祉用具専門相談員にご相談ください。」（あくまで目安であることと相談窓口の存在を、結果が出る前の段階でも明記する。結果画面にも同趣旨の文言を改めて表示する＝結果画面の注記を参照）

### Q1. 車椅子は誰が操作しますか？

| 選択肢 | 遷移先 | 備考 |
|---|---|---|
| 乗る人自身が操作する | Q1-bへ | 自走式グループの入口。ただしこの回答は本人の希望（自己申告）であり、Q1-bで実際の使用シーンに照らして裏取りする |
| 常に介助者が操作する | Q2へ（介助式として） | 介助式グループの入口。ここで確定し、以降は移乗動作の負担だけを聞く |

補足：専門相談員のフィードバックで「介助式と自走式で選ぶポイントは、結局『自分自身で操作するのか、介助者が操作するのか』というところに尽きる」と明言されている（2026-07-17フィードバック）。この段階ではQ1の選択肢そのものに誘導的な文言は加えず、中立な二択として提示する。「自分で動きたい」という自己申告がEC購入者に自走式が偏っている一因になっている可能性がある、という設計上の考慮（設計思想0.）は、Q1-bのステップを設ける根拠として扱い、Q1の見た目・文言には反映させない。

#### Q1の補足チェック：その場でできる身体チェック

自己判断だけでは不安なユーザー向けの補助情報。**新しい分岐は作らず**、Q1の判断材料、および自走式という結果が出た後の付記情報として使う。内容は初版から変更なし。

1. **座ったときのバランス**：背もたれや手の支えなしで座位を保てるか／前後左右にかがんでも自分で戻れるか
   - グラグラして支えが必要なら自走は難しい
2. **手の力**：ペットボトルのフタを開けられる、タオルをしっかり握れる程度の力があるか、左右差がないか
   - 車椅子のこぎ動作は指で握るというより手のひらで押し出す動作が中心。握力は主にブレーキレバー操作にも関わる
3. **腕を動かす力と範囲**：腕を前後に繰り返し動かせるか、肩・肘・手首に痛みが出ないか
   - 押し出す動作ができないと前に進めない。繰り返し動作は肩・肘・手首を痛めやすいことが知られている
4. **繰り返しても疲れないか**：同じ動作を繰り返してもすぐにバテないか
   - ヒアリングでも「100メートル以降はきつい」との指摘あり。持久力がないと屋外での長距離移動は現実的でない

上記に加え、**ブレーキ操作を自分で安全に行えるか**（初版Q2相当）も、独立した分岐にはせず、この身体チェックの注意点として一文で添える。

判定への反映：

| 身体チェックの結果 | 扱い |
|---|---|
| すべて問題ない | Q1「乗る人自身が操作する」としてそのままQ1-bへ |
| どれか1つに不安がある（握力が弱い・すぐ疲れる等） | Q1-bへ進めてよいが、最終的に自走式になった場合は通常タイプより軽量・コンパクトな「TORICO よかセレクト」を優先案内 |
| 座位が不安定／腕をほとんど動かせない・握れない／ブレーキ操作に不安がある | 「常に介助者が操作する」への変更を提案 |

※介福本舗の取扱は自走式・介助式（いずれも手動）のみのため、電動車椅子は案内しない。

### Q1-b. （「乗る人自身が操作する」を選んだ場合のみ）車椅子を使う主な場面は？

Q1-bは補足質問ではなく、本アプリの核心（設計思想0.）にあたる質問。自走式を選んだ人の中に、「自分でこぐ体力はあるが、実際に使う場面は限られていて、普段はある程度自分の足で歩ける」というケースが一定数いる（専門相談員へのヒアリングでの実例を踏まえた本人の解釈）。この場合、使用シーンに自走という動作がほとんど発生せず、実際には軽量な介助式の方が介助者の負担が少なく使いやすいことがある。

初版のQ1-bは選択肢が1パターンしかなく、実際の高齢者の車椅子利用シーンを十分にカバーできていなかった。そのため、頻度（毎日／決まった外出のみ／たまの外出）・場所（屋内外どちらも／屋内のみ）・普段の歩行能力（歩ける／不安がある）という複数の角度から、より具体的な5パターンに再設計した。選択式のみで完結させ、自由記述・AIによる解釈は行わない。

#### 設計判断：ユーザーへの「それでも自走式にしますか？」という再選択はさせない

改訂9まではII（できる）・III・IVで「介助式の方が合うかもしれません」と提示したうえで、「それでも自走式で進める／介助式に変更する」をユーザーに選ばせる設計にしていた。しかし、そもそもこのアプリは「自走式・介助式のどちらが自分に合うか判断がつかない」人が使うものであり、その人に対して「（判断材料を示したうえで）やっぱり自分でどちらか選んでください」と聞き返すのは筋が通らない、という指摘を受けた。実店舗の専門相談員も、ヒアリング後は「あなたはどちらだと思いますか」とは聞かず、集めた情報をもとに自分から結論（多くの場合は介助式）を伝えている。

そのため、II（できる）・III・IVは、ユーザーに選ばせるのではなく、**アプリ側が選択式の回答結果から自動的に介助式と結論づけ**、その理由を結果画面で説明する設計に変更した。ユーザーの操作は選択肢を選ぶことのみで、「診断結果に対してさらに自分で診断し直す」ような場面は作らない。なお、結果画面には理由の説明に加えて「自走式の商品も見る」という商品ページへの補助リンクは残し、結論に納得できない場合でも自走式の商品自体は見られるようにする（これは診断ロジックの分岐ではなく、単なる商品ページへの導線）。

| I. 屋内外問わず、日常的にいろいろな場面で自分の力で動きたい（買い物・散歩・庭先の移動など） | Q2へ（自走式のまま） | 頻度：高い／場所：屋内外どちらも → 自走ニーズが明確 |
| II. 外出はほとんど付き添ってもらうが、自宅の中だけは家族を呼ばずに自分で移動したい | Q1-b-IIへ（室内を車椅子なしで移動できるか確認） | 頻度：日常的（屋内限定）／場所：屋内のみ → 屋内での自立移動ニーズがある（旧版Q3-bに相当）だが、車椅子以外の手段で足りていないかを裏取りする |

### Q1-b-II. （Q1-bで「II. 屋内限定の自走ニーズ」を選んだ場合のみ）自宅の中は、車椅子を使わずに移動できますか？

「屋内だけは自分で動きたい」という回答であっても、実際には杖や伝い歩き、手すり、スロープなどを使えば車椅子なしでも室内を移動できる、というケースがある。この場合、車椅子そのものへの自走ニーズは実は薄く、外出時の付き添いが前提であれば介助式で十分な可能性が高い。III・IVと同じ方針で、ここもユーザーに選ばせず、回答結果からアプリ側が結論づける。

| 選択肢 | 遷移先 | 備考 |
|---|---|---|
| できる（杖・伝い歩き・手すり・スロープなどで、車椅子がなくても室内は移動できる） | Q2へ（介助式として確定） | 車椅子自体の屋内自走ニーズが薄いと考えられるケース |
| できない（車椅子がないと室内移動が難しい） | Q2へ（自走式のまま） | 本来のII「屋内限定の自走ニーズ」に合致するケース。前述のハンドリム幅・小回りに関する注記を表示 |

### Q2. 移乗（車椅子⇄ベッド・トイレなど）は自分で立ち上がってできますか？

自走式・介助式のどちらのグループでも共通の質問。標準型（よかセレクト）と多機能型（よかべスター）の分岐に使う。「立ち上がれる／難しい」という二択だけだと境界があいまいなため、それぞれの選択肢に具体的な状態の例を添えて明確化する（AIレビューでの指摘を反映）。

| 選択肢 | 状態の目安 | 遷移先 | 備考 |
|---|---|---|---|
| 自分で立ち上がって移乗できる | 手すりや車椅子の肘掛けなどにつかまれば、介助者が体を支えなくても立ち上がって移乗できる（見守りだけで足りる） | 標準型（よかセレクト）へ | |
| 立ち上がるのが難しく、移乗が大変 | 介助者が体を支える・抱きかかえるなど、身体的な介助がないと立ち上がれない／移乗できない | 多機能型（よかべスター）へ | 肘跳ね上げ・脚部スイングアウトなどの機構で移乗をサポート |

判断の境界線：「介助者が身体的に支える必要があるかどうか」を基準にする。声かけや見守りだけで済む場合は標準型、体を支える・抱える動作が必要な場合は多機能型、という基準をアプリ上の選択肢の下にも短く添える。

### 結果画面共通の注記（AIレビューを踏まえた追加、改訂11）

本アプリのロジックをAIにレビューしてもらったところ、「自走式が多すぎる」という主要な失敗パターンには効くが、（a）サイズ・設置場所の確認が抜けている、（b）今後の身体状況の変化への視点がない、（c）自己申告に基づく判定である以上「間違えない」とは言い切れない、という指摘を受けた。これらは新しい質問・分岐を増やすのではなく、結果画面の注記で対応する（質問を増やすとQ1〜Q2のシンプルさが崩れるため）。

1. **サイズ・設置場所の確認注記**（自走式という結果が確定した画面に表示）：「ご購入前に、実際に車椅子を使う場所（玄関・廊下の幅など）や、車に載せる場合はトランクのサイズもあわせてご確認ください。」自走式は介助式よりハンドリムの分だけ大きく重くなるため、口コミの「思ったより大きい・重い」を防ぐ目的で自走式の結果画面に表示する。
2. **今後の身体状況の変化に関する注記**（自走式という結果が確定する最後の最後、Q2で標準型／多機能型が決まったあと・商品案内の直前に表示）：「今のところ自走式が合っていても、進行性の病気などで今後身体状況が変わる可能性がある場合は、将来的に介助式への切り替えが必要になることがあります。ご不安な場合は、介助式もあわせてご検討いただくか、福祉用具専門相談員にご相談ください。」質問として選択させるのではなく、自走式という結論が出た人だけに向けた最終確認の一文として位置づける。
3. **「あくまで参考」「心配なら相談を」の明記**：アプリ冒頭のリード文（本章冒頭）に加え、すべての結果画面の末尾にも「この診断結果はあくまで参考です。ご心配な点があれば、福祉用具専門相談員にご相談ください。」を共通で表示する。冒頭と結果画面の両方に置くことで、診断の限界（自己申告に基づく目安であり、断定ではないこと）を利用者が見落とさないようにする。

上記1・2は自走式の結果画面のみに表示し、介助式の結果画面には表示しない（介助式はEC上でのミスマッチ傾向が観測されていないため）。3はすべての結果画面に共通で表示する。

### 任意ステップ（介助式の結果画面のみ）：さらに自分に合うタイプを探す

Q1・Q2は必須フローとして結果（標準型／多機能型）を確定させる。そのうえで、介助式という結果が出た人（標準型・多機能型いずれの場合も）に対して、結果画面に「さらに自分に合う車椅子を探す（任意）」という折りたたみ式の任意ステップを追加する。これは新しい必須分岐ではなく、Q1-bの自走式向け身体チェックと同じ位置づけの補助機能で、選択式（複数選択可のチェックボックス）のみで構成する。

該当する項目にチェックを入れてもらうことで、多機能型（よかべスター 介助式）が向いている理由をより具体的に確認できるようにする。

| チェック項目 | 該当する人の例 | 多機能型（よかべスター）の対応機能 |
|---|---|---|
| ベッドや車から車椅子への移乗が多い | 本人・介助者ともに移乗の機会が多い | 肘掛け跳ね上げ＋脚部開閉により移乗動作がしやすい |
| 円背（背中が丸まっている）や姿勢保持に不安がある | 座位姿勢が崩れやすい・傾きやすい | 背張り調整機能で楽な姿勢を保てる |
| 介助者に腰痛など身体的負担がある | 押す・移乗させる動作で介助者の腰に負担がかかっている | 後輪が座面より低い介助式の構造により、押す・移乗させる動作の負担が軽い |
| 雨天や屋外での使用が多い | 施設や家庭で屋外・悪天候時の利用機会が多い | ドラムブレーキにより雨天でも安定して効く |
| 外出頻度が高く、長時間座ることが多い | 頻繁に外出し、車椅子に長く座る時間がある | クッション性・各種調整機能により疲れにくい |

反映ロジック：

- チェック数が0件：Q2の結果（標準型／多機能型）をそのまま最終案内とする。
- 1件以上チェックがあり、Q2の結果がすでに多機能型：チェックした項目を「あなたに多機能型が向いている理由」として結果画面に一言ずつ添え、納得感を補強する。
- 1件以上チェックがあり、Q2の結果は標準型：「移乗のしやすさ以外にも当てはまる点があるようです。多機能型（よかべスター 介助式）も候補に加えてご検討ください」と提示し、標準型・多機能型の両方を比較できる形で案内する（Q2の結果を強制的に上書きはせず、選択肢を提示するにとどめる）。

※各項目の機能説明は、実際のよかべスター介助式の商品ページ記載内容と齟齬がないか、アプリ化の際に商品ページで最終確認すること。

## 7. 用途別事例（自由記述の代わりに提示する参考ケース）

自由記述・AIによる解釈を使わない代わりに、専門相談員のフィードバックにあった「用途別の事例」を結果画面や補足に掲載し、個別事情のあるユーザーが自分に近いケースを参考にできるようにする。

| ケース | 状況 | 案内 | 対応するQ1-b選択肢 |
|---|---|---|---|
| 屋内外問わず日常的に自分で動きたい | 買い物・散歩・通院など幅広い場面で自分の力で移動したい | そのまま自走式（移乗の負担に応じて標準型／多機能型） | I |
| 屋内だけは自分で移動したい（車椅子が本当に必要） | 外出は付き添ってもらうが、自宅内は杖や伝い歩きでは難しく、車椅子がないとトイレ・居室間などを移動できない | そのまま自走式。ハンドリム分の幅で屋内の取り回しがしにくい点を注記 | II→Q1-b-II「できない」 |
| 屋内は歩けるが車椅子でも動きたいと答えていた | 自宅内は杖や伝い歩き、手すり・スロープなどで実際には移動できる。外出時は付き添い前提 | 車椅子自体の屋内自走ニーズは薄いと考えられるため、軽量な介助式（よかセレクト 介助式）を提案 | II→Q1-b-II「できる」 |
| 通院メインで自走したいが介助者の負担も減らしたい | 自分で漕ぐ体力はあるが、車椅子を使うのは通院・デイサービスの送迎など決まった外出の時だけ。普段はゆっくりなら歩ける | 主目的である通院介助をやりやすくすることを優先し、軽量な介助専用車椅子（よかセレクト 介助式）を提案 | III |
| たまの外出で家族と一緒に使う | 買い物・外食・旅行など、たまの外出のときだけ使う予定。普段の歩行には不安がある | 外出時は家族が付き添う前提になりやすいため、軽量な介助式（よかセレクト 介助式）を提案 | IV |
| 移乗の負担が大きい | 自走・介助いずれでも、車椅子からベッドやトイレへの立ち上がりが難しい | 標準型ではなく多機能型（よかべスター）を提案し、肘跳ね上げ・脚部スイングアウトで移乗をサポート | 共通（Q2） |
| たまの外出・旅行で、念のため車椅子を持っていきたい | 普段は歩けるが、旅行など長距離の外出時だけ不安があり車椅子を用意したい（長距離移動そのものには使わない） | よかセレクト 介助式を基本としつつ、より軽量な旅ぐるま KA6も参考として提示（長距離移動・日常使用には不向きなため、それ以外のケースでは積極的に案内しない） | IV |

## 8. 制約・注意事項

- 介福本舗の取扱は自走式・介助式（いずれも手動）のみ。電動車椅子は取扱なしのため、診断結果・補足のいずれにも登場させない。
- 本アプリは静的なWebページとして運用し、AI・バックエンドとの連携は行わない。そのため使用シーンの確認（Q1-b）は選択式のみで構成し、自由記述欄は設けない。選択式でカバーしきれない個別事情は、事例紹介（第7章）で補う。
- Q1補足の身体チェックはあくまで自己チェックの目安であり、医療的診断ではない旨をアプリ上に明記することが望ましい。この身体チェック（座位バランス・手の力・腕の力と可動域・疲労耐性）が、自走式を利用できる状態かどうかの具体的な判断基準にあたる。
- 「工夫すれば使える」という中間評価は、既存のフローに新しい分岐として組み込まず、自走式という結果が出たあとの商品案内（軽量タイプの優先）として扱う。
- 本アプリはQ1・Q2とも自己申告（本人・家族の回答）に基づく判定であり、実際の身体状況を測定するものではない。「間違いをかなり減らせるが、間違えないとは言い切れない」という限界を前提に、結果画面共通の注記（4章）と身体チェック（4章Q1補足）で補うが、最終的な購入判断はユーザー自身・家族・専門相談員に委ねる。
